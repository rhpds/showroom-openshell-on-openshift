= OpenShell and OpenClaw on Red Hat OpenShift
:navtitle: Home

Your team needs to evaluate supervised AI agent execution on OpenShift — running agent workloads in isolated sandboxes with configurable security policies.
This lab walks you through every layer of that stack: installing the infrastructure, connecting an agent application, and testing what security policies actually enforce.

You work on a real OpenShift {ocp_version} cluster with cluster-admin access.
Each module ends in a verified, stable state, so you can stop after any module and return later.

== What you'll learn

* Install Agent Sandbox controller and OpenShell gateway on OpenShift {ocp_version}
* Deploy OpenClaw and configure delegated execution into supervised sandboxes
* Apply filesystem, network, and process security policies to sandbox pods
* Compare deny-all, managed, and allowlist policy profiles under controlled conditions

== Who this is for

This lab is for platform engineers and cluster administrators who are evaluating whether supervised AI agent workloads fit their OpenShift environment.
You should be comfortable working with `oc`, Helm, and Kubernetes custom resources.

*Experience level*: Intermediate

== Prerequisites

Before starting, confirm you have all of the following:

* Access to an existing OpenShift {ocp_version} cluster
* Cluster-admin privileges
* `oc`, `kubectl`, and Helm installed locally with a kubeconfig targeting the cluster
* Permission to use an LLM API credential for the OpenClaw modules (Module 2 onward)

Cluster provisioning and obtaining cluster-admin access are outside the scope of this lab.

== Security notice

[WARNING]
====
This lab is an experimental proof of concept, not production security guidance.
Some modules use the `privileged` SCC for sandbox pods, disable gateway TLS, and permit unauthenticated gateway access.
These settings exist to keep the exercises repeatable on a test cluster.
Run this lab only on a private, disposable evaluation cluster.
Do not expose the OpenShell gateway Service or its port-forward to untrusted clients.
====

The supervised agent process still receives OpenShell's process, filesystem, network, and inference controls.
Those controls do not make a privileged, plaintext, unauthenticated configuration safe for production.
Read each warning before applying a configuration, and do not treat any lab setting as a recommended production posture.
